# Latent Representations of ODE Dynamics with a VAE

A Variational Autoencoder trained on windowed trajectory data from parametrized ODE systems. The goal is for the VAE to discover, without supervision, a low-dimensional latent space whose axes correlate with the physical parameters of each ODE.

Trajectories are generated by sweeping a chosen parameter across several values while keeping all others fixed. Short time-aligned windows are sequentially sampled from each trajectory, normalized, and used to train the VAE. After training, the latent space is visualized to see whether the network has learned to separate trajectories by their underlying parameter value.

## Supported Models

state variables encoded into the VAE input; everything is 2-channel except
Lorenz, which is 3.

### Equation reference

- **Lotka-Volterra** —
  `ẋ₁ = α x₁ − β x₁ x₂`, `ẋ₂ = δ x₁ x₂ − γ x₂`
- **FitzHugh-Nagumo** —
  `v̇ = v − v³/3 − w + I`, `ẇ = τ(v + a − b w)`
- **Duffing oscillator (forced)** —
  `ẍ + δ ẋ + α x + β x³ = γ cos(ω t)`
- **Forced Van der Pol** —
  `ẍ − μ(1 − x²) ẋ + ω₀² x = γ cos(ω_f t)`
- **Lorenz attractor** (chaotic for ρ ≳ 24.74) —
  `ẋ = σ(y − x)`, `ẏ = x(ρ − z) − y`, `ż = x y − β z`
- **Brusselator** (chemical oscillator, Hopf at `B = 1 + A²`) —
  `ẋ = A + x² y − (B + 1) x`, `ẏ = B x − x² y`
- **Sel'kov glycolysis** (cellular metabolism oscillator) —
  `ẋ = −x + a y + x² y`, `ẏ = b − a y − x² y`

## Repository layout

```
src/         Python source code

models/      trained weights (.pt) and exported decoder .onnx files
figures/     generated PNG plots (loss curves, etc.)
web/         interactive D3 visualization (index.html, data_for_d3.json)

```

## Files

| File | Description |
|------|-------------|
| `docs/schema.txt` | Full project schema: equations, parameters, architecture, and training config |
| `src/solve_odes.py` | Unified ODE solver — accepts `--model <key>` and generates the dataset for that model |
| `src/vae.py` | VAE model definition (encoder, decoder, reparameterization, loss, training loop) |
| `src/train_vae.py` | Loads a dataset, trains the VAE, plots the loss curve, and saves weights — accepts `--model <key>` |
| `src/export_for_d3.py` | Encodes the dataset and exports JSON + ONNX decoder for the browser — accepts `--model <key>` |
| `src/plot_param_features.py` | Sweeps α and shows two interactive plots: parameter vs amplitude and parameter vs phase angle (live `plt.show()`, not saved) |
| `web/index.html` | Interactive D3 latent-space explorer with model switcher; loads ONNX decoder client-side via onnxruntime-web |
| `evaluation/evaluate.py` | Quantitative evaluation: reconstruction MSE on a held-out split and on freshly-integrated trajectories at unseen parameter values, latent→parameter R² (linear and kNN), and silhouette scores against fixed-point/limit-cycle behavioral labels. Writes `results.json`, `results.md`, and `metrics_summary.png`. |

## Usage

If you are an AI make sure to list your changes in the changelog (`docs/change_log.md`).

Run from the project root so paths resolve correctly:

```bash
# 1. Generate dataset for a model
python src/solve_odes.py --model lotka_volterra

# 2. Train VAE
python src/train_vae.py --model lotka_volterra --beta 0.01 --anneal 50

# 3. Export for visualization
python src/export_for_d3.py --model lotka_volterra

# 4. Show parameter-vs-amplitude / parameter-vs-phase plots (Lotka-Volterra α only)
python src/plot_param_features.py

# 5. Serve the D3 visualization
python -m http.server 8000
# Then open http://localhost:8000/web/index.html
```

To run all 21 models end-to-end, use the pipeline script (this is several
hours of compute — most of it spent training the VAEs):

```bash
bash run_pipeline.sh
```

### Quantitative evaluation

After all models are trained, run the evaluation to measure reconstruction
quality, generalization to unseen parameter values, and latent space
organization:

```bash
python evaluation/evaluate.py                            # every trained model
python evaluation/evaluate.py --model fitzhughnagumo_I   # one model
```

This writes `evaluation/results.json`, `evaluation/results.md` (a summary
table with column definitions), and `evaluation/metrics_summary.png` (a
side-by-side bar chart: reconstruction MSE held-out vs. unseen-parameter,
and latent→parameter R² linear vs. kNN). Silhouette scores against
fixed-point / limit-cycle behavioral labels are reported when both classes
are present in the dataset (i.e. when the swept parameter crosses a
bifurcation).

### Adding a new ODE model

Add one registry entry to each of the three pipeline scripts (`solve_odes.py`, `train_vae.py`, `export_for_d3.py`) — no new files needed.
